Prompt
需求定義：使用 CRISP-DM 框架分析 Titanic 資料集，包括：

從網路獲取資料。
建立混淆矩陣和計算準確度。
將模型部署並上傳至 Kaggle。
資料集：使用 GitHub 上的 Titanic 資料集（包含 train.csv 和 test.csv）。

模型目標：初始模型準確度為 0.76555，目標是提升至超過 0.76555。

過程
數據獲取：

使用 pandas 從 GitHub 下載 Titanic 的訓練和測試資料集。
數據前處理：

進行數據清理，包括填補缺失值和轉換類別變數。
引入新特徵（如 FamilySize 和 Title）以提升模型的解釋能力。
模型選擇與訓練：

初步使用 Logistic Regression 模型進行預測。
使用隨機森林（Random Forest）模型作為更複雜的替代方案，進行更準確的預測。
超參數調整：

使用 GridSearchCV 進行網格搜尋以找到最佳超參數配置，設定了多個參數範圍。
確認最佳參數後訓練模型。
模型評估：

使用混淆矩陣和準確度評估模型表現。
計算準確度並與目標分數進行比較。
模型優化：

由於網格搜尋的速度較慢，建議考慮以下優化方法：
縮小參數範圍或使用隨機搜尋（RandomizedSearchCV）。
減少交叉驗證的次數以加快運算。
減少訓練數據的大小來快速測試參數組合。
嘗試其他模型或簡化模型以提高計算效率。
結果部署：

預測測試數據並生成提交檔案（submission.csv），準備上傳至 Kaggle 進行評分。
